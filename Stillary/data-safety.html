Play Console Reference

Stillary Data Safety

Use this page as the source of truth when filling in App content > Data safety in Google Play Console. It reflects the current release implementation: photos stay on device, while Firebase Analytics and Crashlytics collect limited technical telemetry.

Data Type Status What This Means For Stillary Purpose Shared
Photos and videos Not collected Photo content is processed locally on device. Stillary does not transmit imported photos, memo text, or exported posters to its own servers. Local-only processing No
App activity Collected Firebase Analytics records anonymous screen-view events when the Anonymous analytics switch is enabled. Analytics No
App info and performance Collected Firebase Crashlytics receives crash logs and diagnostics from release builds to improve reliability. App functionality, Analytics No
Device or other IDs Collected Firebase services may process app instance or device identifiers needed to support analytics and diagnostics. App functionality, Analytics No

Additional Answers

  • Data is encrypted in transit: Yes for telemetry sent to Firebase.
  • User data is sold: No.
  • User data is used for advertising: No.
  • Account creation required: No.

User Controls

  • The Settings screen includes an Anonymous analytics toggle.
  • Turning the toggle off stops screen-flow analytics going forward.
  • Crash diagnostics remain enabled in the current release build.

Do Not Mark As Collected

  • Location
  • Personal info
  • Financial info
  • Messages
  • Files and docs
  • Audio files
  • Web browsing
